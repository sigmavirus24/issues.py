issues.py

issues.py will be a small script which will allow you to track issues on any projects you'd like from the command line.

Access

You'll be able to access the GitHub API anonymously so you don't have to store your username and password. In fact I'm strongly considering only allowing anonymous access to avoid people placing their usernames and passwords in a plain text file.

Config file

While the main part of the script is done, I have yet to decide exactly what the config file will look like but it should look something like:

[repo_owner0]
repository0
repository1

[repo_owner1]
repository2
repository3
repository4

[repo_owner2]
repository5

A specific example would be along the lines of:

[sigmavirus24]
Todo.txt-python
issues.py
github3.py

[kennethreitz]
clint
requests
httpbin
